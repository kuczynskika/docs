# Labels

The **Labels** settings page is where you manage the **shared label library** for your workspace—the catalog of labels that everyone on your team can apply to conversations.

<Note>
  Looking for how to **apply** labels inside a conversation, build label-based filters, or rename labels from the inbox? See [Inbox → Labels](/inbox/labels). This page focuses on **managing the label library** from Settings.
</Note>

---

## Open the Labels page

Go to **Account → Settings → Workspace Settings → Labels**.

---

## Create a new label

1. Click **Create label**.
2. Set the label's properties:
   - **Name** — short and scannable (e.g., `Support: Urgent`, `Region: West`).
   - **Color** — pick a color to help teammates spot it at a glance.
   - **Icon** — choose an icon that fits the label's purpose.
3. Click **Save**.

The label becomes available everywhere labels are used—conversation label menus, filters, and views.

<Tip>
  Use short prefixes like `HR:`, `Sales:`, or `Support:` to group related labels. Since labels sort alphabetically, prefixes are an easy way to control where labels appear in lists.
</Tip>

---

## Edit a label

1. Find the label in the list (or search by name).
2. Click the label to open its editor.
3. Update the **name**, **color**, or **icon**.
4. Click **Save**.

<Warning>
  Renaming a label updates it **everywhere** it's already applied. Existing conversations keep the relationship to the label—only the displayed name/color/icon changes.
</Warning>

---

## Delete labels

You can delete a single label or multi-select to delete several at once.

**Single label:**

1. Find the label in the list.
2. Click the **delete** icon.
3. Confirm.

**Bulk delete:**

1. Multi-select the labels you want to remove.
2. Click **Delete selected**.
3. Confirm.

Deleting a label removes it from every conversation it was applied to.

---

## Label sort order

<Note>
  Labels are sorted **alphabetically (A–Z)** by name across label menus, settings, and filters. To influence where a label appears, rename it with a prefix (e.g., `HR:`, `Support:`).
</Note>

---

## Who can manage labels

Access to label management is controlled by your role:

| Role         | Labels access |
| ------------ | ------------- |
| **Admin**    | Owner (create, edit, delete) |
| **Manager**  | Editor (create, edit, delete) |
| **Operator** | Editor (create, edit, delete) |

See [Roles & Permissions](/settings/workspace-settings/team-management) for the full default permissions table and how to customize roles.
